Manipulating Data

Getting Variables

In JBON there are two ways to parse text. You can either open up a file with JBON text, or use the parser to parse a string. To start, lets begin talking about opening a file. All jbon files are proceeded with the .jbon file extension. To load a file, either call a jbon file constructor with the filepath, or load the data at a later time. 


jbon::File file("test.jbon")
//or 
jbon::File file();
file.load("test.jbon");

As you can see, everything in the jbon library is under the jbon namespace, so for convinince sake, using the namespace would not be a bad idea. JBON files can also include jbon files, which is different than loading them. Think about it like c++ header that you're including. If you find yourself often copying the same class definition, you can define the class in a jbon file once and include it in any other file loading. This was added to reduce redundancy, so don't hesitate to use it. Also, keep in mind, including a jbon file will only look at the classes and not load the data. 


jbon::File file;
file.include("CatHeader.jbon");
file.load("CatData.jbon");

Setting Variables

The other way to parse JBON text is to use the parse function. Pass the text you want parsed and it will return a ClassCollection. 


auto collection = jbon::parse("Coord{x, y} Coord(4,2)");

Making Objects

The only data that can be saved to a file is data opened with a JBON file. The user, however, can save any data manually with the serialize functions provided by nearly all classes. To save data opened with JBON file, simply call save(). 


jbon::File file;
file.load("data.jbon");
auto& collection = file.getClassess();
//change the data refrenced by the collection
file.save();
